Project Overview

What this project is and what it tests

This is an end-to-end (E2E) automation testing project built using Playwright — a modern, Microsoft-developed browser automation framework. The project tests the web application hosted at https://raya-business-owner-frontend.vercel.app/. It follows the Page Object Model (POM) design pattern to keep tests clean and maintainable.

🎯 Application Under Test

Raya Business Owner Frontend — a SaaS platform. The tests cover user Signup (full multi-step flow) and user Login.

🛠️ Tech Stack

Playwright v1.61 JavaScript (ESM) Node.js Chromium Browser

💡

E2E Testing means we test the application the same way a real user would — by opening a real browser, clicking buttons, filling forms, and verifying results.

Architecture: Page Object Model (POM)

The design pattern used in this project

POM is a design pattern in test automation where you create a separate class for each page of the application. That class holds all the locators (selectors that find elements) and actions (methods like .click(), .fill()) for that page. Test files then use these classes — they don't contain raw selectors themselves.

→

→
BrowserChromium
→
Web Appraya-business…

✅ Why POM?

If a button's selector changes, you fix it in one place (the Page Object) instead of searching through every test file. Makes tests maintainable, readable, and reusable.

❌ Without POM

Selectors scattered in every test file. One UI change breaks 10 tests. Hard to read. Hard to maintain. Not professional.

import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: './tests',            // Where to look for test files
  fullyParallel: true,           // Run all tests at the same time (faster)
  forbidOnly: !!process.env.CI,  // Fail build if test.only() left in code (CI safety)
  retries: process.env.CI ? 2 : 0, // On CI: retry failed tests 2 times; locally: 0
  workers: process.env.CI ? 1 : undefined, // On CI: 1 parallel worker; locally: auto
  reporter: 'html',              // Generate an HTML report after tests finish

  use: {
    trace: 'on-first-retry',     // Record a trace only when test is retried (debugging)
  },

  projects: [
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] }, // Spread operator: copies Chrome device settings
    },
  ],
});
OptionExplanation
defineConfig() A helper function from Playwright that enables type-safety and autocomplete for the config object. Not strictly required but best practice.
devices A pre-built collection of device profiles (screen size, user agent) for Chrome, Firefox, Safari, iPhone, Pixel, etc.
testDir Playwright scans this folder for files ending in .spec.js, .test.js, etc.
fullyParallel All test files run at the same time using multiple browser instances. Speeds up the suite significantly.
process.env.CI process.env is a Node.js object containing environment variables. CI is set to "true" automatically by GitHub Actions, Jenkins, etc.
!! Double NOT operator. Converts any truthy/falsy value to a strict true or false boolean. e.g. !!"hello" = true, !!undefined = false.
trace: 'on-first-retry' Records a Playwright Trace (network, DOM snapshots, screenshots) only when a test fails and is retried. View with npx playwright show-trace.
...devices['Desktop Chrome'] The spread operator (...) copies all properties from the Chrome device object into the use object. It's like copy-pasting settings.
reporter: 'html' After tests finish, Playwright generates a rich HTML report. Open with npx playwright show-report.

Pages/login.js — LoginPage Class

Page Object for the Login page

Pages/login.js

Page Object Model 
export class LoginPage {                               // ← Line 1: Named export of a class

    constructor(page) {                                // ← Runs when: new LoginPage(page)
        this.page = page;                              // ← Save page reference on the instance
        this.email_txtbox = page.getByRole('textbox', { name: 'Enter your email' });
        this.pass_txtbox  = page.getByRole('textbox', { name: 'Enter your password' });
        this.login_btn    = page.getByRole('button',  { name: 'Login' });
    }                                                  // ← All locators stored as instance properties

    async login(email, password) {                     // ← async: function returns a Promise
        await this.email_txtbox.click();               // ← wait for click to complete
        await this.email_txtbox.fill(email);           // ← type the email parameter into the field
        await this.pass_txtbox.click();
        await this.pass_txtbox.fill(password);
        await this.login_btn.click();
    }

    async gotoLoginPage() {
        await this.page.goto("https://raya-business-owner-frontend.vercel.app/");
    }
}
Keyword / MethodDetailed Explanation
export Makes this class available to be imported in other files. Without export, the class is private to this file only.
class A blueprint for creating objects. Defines what properties and methods the object will have. In OOP terms: a class is a template, an object is an instance of that template.
constructor(page) A special method called automatically when you do new LoginPage(page). Used to initialize the object's properties. The page parameter is the Playwright Page object passed in.
this Refers to the current instance of the class. this.page = page saves the page reference so other methods can use this.page later.
async Marks a function as asynchronous. It will always return a Promise. Required when using await inside the function. Playwright actions are async because they involve network/browser operations.
await Pauses execution until the Promise resolves. Without await, the browser action starts but the code moves to the next line immediately — causing race conditions and errors.
getByRole('textbox', {name:...}) Finds an element by its ARIA role. 'textbox' finds input fields. The name option matches the accessible label of the field. This is the recommended Playwright locator because it mimics how users interact with UI.
getByRole('button', {name:...}) Finds a button element by its visible text or ARIA label. More reliable than CSS selectors because it doesn't break when styling changes.
.click() Simulates a mouse click on the element. Playwright auto-waits for the element to be visible and enabled before clicking.
.fill(value) Clears the input field and types the given value. More reliable than .type() because it sets the value directly.
.goto(url) Navigates the browser to the given URL. Waits until the page is in the 'load' state by default.

export class signup {

    constructor(page) {
        this.page = page;
        this.nameField   = page.getByRole('textbox', { name: 'Enter your name' });
        this.emailField  = page.getByRole('textbox', { name: 'Enter your email' });
        this.aggreeterms = page.locator('.trust-checkbox').first();  // CSS class selector
        this.signupbtn   = page.getByRole('button', { name: 'Sign Up' });
        this.otp1 = page.locator('#otp-0');  // ID selector #id
        this.otp2 = page.locator('#otp-1');
        this.otp3 = page.locator('#otp-2');
        this.otp4 = page.locator('#otp-3');
        this.otp5 = page.locator('#otp-4');
        this.otp6 = page.locator('#otp-5');
        this.verifyotp = page.getByRole('button', { name: 'Verify OTP' });
    }
signuptoWeb — step-by-step flow
    async signuptoWeb(name, email, otp) {
        // ── STEP 1: Initial Signup Form ────────────────────────────
        await this.nameField.fill(name);          // fill name field with parameter
        await this.emailField.fill(email);        // fill email field with parameter
        await this.aggreeterms.click();           // tick the T&C checkbox
        await this.signupbtn.click();             // click Sign Up → sends OTP to email

        // ── STEP 2: OTP Verification (6 separate boxes) ────────────
        await this.otp1.fill(otp);  // otp = "123456" → fills '1' into box 1
        await this.otp2.fill(otp);  // Note: each box gets the FULL string
        await this.otp3.fill(otp);  // (app internally reads first char of each)
        await this.otp4.fill(otp);
        await this.otp5.fill(otp);
        await this.otp6.fill(otp);
        await this.verifyotp.click();  // submit OTP

        // ── STEP 3: Terms & Plan Selection ─────────────────────────
        await this.aggreeterms.first().click();
        await this.page.getByRole('button', { name: 'Continue' }).click();
        await this.page.getByRole('button', { name: 'Select this plan' }).nth(1).click();
        // .nth(1) = second matching element (0-indexed)

        // ── STEP 4: Billing / Payment Form ─────────────────────────
        await this.page.getByRole('textbox', { name: '0000 0000 0000' }).fill('0000000000000000');
        await this.page.getByRole('textbox', { name: '/00' }).fill('11/50');       // expiry
        await this.page.getByRole('textbox', { name: '000', exact: true }).fill('123'); // CVV
        await this.page.getByRole('button', { name: 'Pay now' }).click();

        // ── STEP 5: Password Creation ───────────────────────────────
        await this.page.getByRole('textbox', { name: 'Enter password', exact: true }).press('CapsLock');
        await this.page.getByRole('textbox', { name: 'Enter password', exact: true }).fill('Test@123');
        await this.page.getByRole('button', { name: 'Create' }).click();

        // ── STEP 6: Business Profile Setup ─────────────────────────
        await this.page.getByRole('textbox', { name: 'Enter your business name' }).fill('qaTech');
        await this.page.locator('.dropdown-selected').first().click();
        await this.page.getByText('Spa', { exact: true }).click();
        await this.page.locator('div:nth-child(6) > .common-dropdown > .dropdown-selected').click();
        await this.page.getByText('Pacific Time - US & Canada').click();
        // CSS child combinator: div:nth-child(6) > .common-dropdown > .dropdown-selected

        // ── STEP 7: Address ────────────────────────────────────────
        await this.page.getByRole('textbox', { name: 'Enter Address' }).fill('indore');
        await this.page.getByText('United States').click();
        await this.page.getByText('Alabama').click();
        await this.page.getByRole('textbox', { name: 'Enter Zip Code' }).fill('456776');
        await this.page.getByRole('textbox', { name: 'Ex. MangoMint' }).fill('aropay');
        await this.page.getByRole('button', { name: 'Continue' }).click();

        // ── STEP 8: Training System Selection ──────────────────────
        await this.page.getByText('Build Your Training System...').click();
        await this.page.getByRole('button', { name: 'Continue' }).click();
    }

Keyword / MethodExplanation
locator('.trust-checkbox') CSS class selector. The dot . means "find element with class name trust-checkbox". Equivalent to CSS .trust-checkbox.
locator('#otp-0') CSS ID selector. The hash # means "find element with id otp-0". IDs are unique per page, so this is very precise.
.first() When a locator matches multiple elements, .first() picks the very first one (index 0).
.nth(1) Picks the element at index 1 (second element, since indexing starts at 0). .nth(0) = same as .first().
.press('CapsLock') Simulates pressing a keyboard key. Here used to toggle CapsLock before typing a password. Other examples: 'Enter', 'Tab', 'Escape'.
getByText('Spa', { exact: true }) Finds an element by its visible text content. exact: true means exact match (case-sensitive). Without exact, it matches text that contains "Spa".
div:nth-child(6) > .dropdown CSS child combinator (>). Finds .dropdown that is a direct child of the 6th div. More specific than a space (descendant) selector.
locator('div').filter({ hasText: /^Address$/ }) .filter() narrows down matched elements. hasText accepts a regex. /^Address$/ is a regex meaning: starts with ^ and ends with $ "Address" — exact match.
locator('svg').nth(1) Targets SVG elements (icons) by their tag name. nth(1) picks the second SVG (e.g., a toggle icon for show/hide password).
{ name: '000', exact: true } The exact option forces exact label matching. Without it, a field labeled "0000 0000 0000" would also match '000'.

    const login = new LoginPage(page);

KeywordExplanation
import { test, expect } ES Module named import. Pulls only test and expect from the Playwright package. test registers a test. expect is used for assertions (not used in this file yet).
import { LoginPage } from '../Pages/login' ../ goes up one folder (from tests/ to the root), then into Pages/. The curly braces {} indicate a named import (because we used export class, not export default).
test(name, callback) Registers a test case. The first argument is the display name. The second is an async function that contains the test steps.
async ({ page }) => {} An async arrow function. { page } is destructuring — Playwright passes a fixtures object; we extract only page from it. page is a fresh browser page for each test.
const Declares a block-scoped variable that cannot be reassigned. Use const when the variable reference won't change. let when it will. Never use var in modern JS.
new LoginPage(page) new calls the constructor of the class and creates an instance (object). The page fixture is passed so the Page Object can interact with the browser.
.spec.js The .spec. suffix is a naming convention meaning "specification test". Playwright automatically discovers files matching *.spec.js, *.test.js, etc.

Locator Strategies

How Playwright finds elements on the page

A locator is how Playwright identifies and interacts with an HTML element. Choosing the right locator makes tests robust and less likely to break when the UI changes.

StrategyExample in CodeWhen to UseReliability
By Role getByRole('textbox', {name: 'Email'}) ✅ Always preferred. Mimics how screen readers see the page. ⭐⭐⭐⭐⭐
By Text getByText('Sign Up') Good for buttons/links with visible text. ⭐⭐⭐⭐
CSS Class locator('.trust-checkbox') When no ARIA role/label is available. Can break if CSS changes. ⭐⭐⭐
CSS ID locator('#otp-0') When element has a stable unique ID. IDs are unique per page. ⭐⭐⭐⭐
CSS Child locator('div:nth-child(6) > .dropdown') Last resort for complex nested structures. Fragile. ⭐⭐
SVG Tag locator('svg').nth(1) For icon elements that have no text/role. Very fragile. ⭐
🏆

Interview Tip: Always say "I prefer getByRole() over CSS selectors because it's more aligned with accessibility standards and is less likely to break when the UI is restyled."

Interview Q&A Bank

Top questions with model answers based on this project

What is Playwright? Why use it over Selenium?
Playwright is a Microsoft-developed open-source browser automation framework for Node.js, Python, Java, and .NET. Unlike Selenium, Playwright supports auto-waiting (no explicit waits needed), runs multiple browsers (Chromium, Firefox, WebKit) in a single API, supports parallel execution out of the box, and has a built-in trace viewer and HTML reporter. It also supports network interception and mobile emulation natively.
What is Page Object Model (POM) and why do you use it?
POM is a design pattern where each page of the application has a corresponding class. This class stores all element locators and action methods for that page. Benefits: (1) Reusability — write the locator once, use it in many tests. (2) Maintainability — if a selector changes, update one file. (3) Readability — tests read like plain English, not selector soup.
What is async/await? Why is it needed in Playwright?
async/await is JavaScript's syntax for handling Promises (asynchronous operations). Playwright actions like page.goto(), .click(), .fill() are async because they involve browser network operations that take time. Without await, the next line runs immediately before the browser action completes — causing race conditions and failures. async marks the function as asynchronous; await pauses execution until the Promise resolves.
What is the difference between export and export default?
export class Foo is a named export — imported with curly braces: import { Foo } from './file'. A file can have multiple named exports. export default class Foo is a default export — imported without braces: import Foo from './file'. A file can have only one default export. This project uses named exports (export class LoginPage, export class signup).
What is getByRole() and why is it preferred?
getByRole() locates elements by their ARIA role (e.g., 'button', 'textbox', 'heading') and accessible name. It's preferred because: (1) It's accessibility-first — tests what users actually see. (2) It's resilient to styling/CSS changes. (3) Playwright recommends it as the first locator strategy. Compare to CSS selectors which break if a class name changes.
What does .first() and .nth() do?
When a locator matches multiple elements on the page, .first() selects the first element (index 0) and .nth(n) selects the element at index n (zero-based). For example, page.locator('.trust-checkbox').first() selects the first checkbox with class trust-checkbox. Without these, Playwright throws an error if more than one element is matched.
What is the constructor in a class?
The constructor is a special method that runs automatically when you create an instance with new. It initializes the object's properties. In this project, the constructor takes a page parameter and stores all the element locators as this.propertyName so they can be used in other methods of the class.
What is playwright.config.js and what are its key options?
It's the global configuration file for Playwright. Key options: testDir — where test files are. fullyParallel — run all tests simultaneously. retries — retry failed tests N times. reporter — how to report results ('html', 'json', 'list'). projects — which browsers to test on. use.trace — when to record execution traces.
What is process.env.CI and why is it used in the config?
process.env is a Node.js object containing operating system environment variables. CI is a variable automatically set to "true" by CI/CD platforms like GitHub Actions, Jenkins, and CircleCI. By checking process.env.CI, the config behaves differently in CI (e.g., more retries, 1 worker, fail on test.only) vs. local development.
What is the difference between .fill() and .type()?
.fill(value) clears the input and sets the value directly — it's faster and more reliable. .type(value) simulates actual key presses character by character — slower but useful when testing key-event listeners. For most E2E tests, .fill() is preferred. Playwright's .fill() also dispatches input/change events properly.
What is the spread operator (...) in the config?
...devices['Desktop Chrome'] — the spread operator copies all properties of the Chrome device object (viewport width, height, userAgent, etc.) into the use object. It's equivalent to manually listing every property. This allows you to add extra settings alongside the device preset without modifying the original object.
How do you run only one test file? How do you run tests in headed mode?
Run a specific file: npx playwright test login.spec.js. Run in headed mode (see the browser): npx playwright test --headed. Run a specific test by name: npx playwright test -g "User should able to Login". Open interactive UI mode: npx playwright test --ui. Show last HTML report: npx playwright show-report.
What is a fixture in Playwright (e.g., the page object in tests)?
Fixtures are built-in helpers Playwright provides to each test. The most common is page — a fresh browser page created before each test and destroyed after. Others include browser, context, request. They're passed via destructuring: async ({ page }) => {}. Playwright handles setup/teardown of fixtures automatically, ensuring test isolation.
What is devDependencies vs dependencies in package.json?
dependencies: packages needed in production (e.g., Express, React). devDependencies: packages only needed during development/testing (e.g., Playwright, Jest, ESLint). When deploying to production with npm install --production, devDependencies are skipped. Playwright goes in devDependencies because end users don't run tests.
What is the this keyword and why is it important in classes?
this refers to the current instance of the class. In the constructor, this.page = page stores the Playwright page on the instance so other methods like login() can access it via this.page. Without this, each method would have no shared state — it would be a standalone function, not part of an object.
